Klene

Safe cleanup for Linux and Windows. Klene gives you a clean desktop app and a fast CLI for reviewing cleanup opportunities before anything gets removed.

View on GitHub Preview first GUI and CLI

Why I made Klene

Klene started as an Arch Linux cleanup tool because that was the first system I wanted to support deeply. It now expands carefully into other Linux distributions and Windows without dropping the preview-first, confirmation-first safety model.

What it cleans

Arch Linux and Arch-like

Pacman cache, orphan packages, AUR cache, Flatpak, journal, trash, thumbnails, and low-risk user cache.

Debian and Ubuntu family

APT cache, apt autoremove preview, Snap cache review, and the shared Linux cleanup set.

Fedora

DNF cache, DNF autoremove preview, plus the shared Linux cleanup set.

openSUSE

Zypper cache plus the shared Linux cleanup set.

Generic Linux

Trash, thumbnails, low-risk user cache, journal, and Flatpak when available.

Windows

User temp, Windows temp, Recycle Bin, thumbnail cache, and Windows error reports with conservative safety limits.

Safety-first design

  • Preview is always non-destructive.
  • Real cleanup requires confirmation.
  • Package removal gets extra care.
  • Protected paths are refused.
  • Registry cleanup, browser data cleanup, and arbitrary path cleanup stay blocked.
  • The same backend powers both the GUI and CLI.

GUI and CLI

Klene is built for both kinds of users. If you want a polished desktop utility, the PySide6 GUI gives you a clear dashboard, safe previews, and branded desktop integration. If you prefer the terminal, the Typer CLI gives you scan, doctor, about, and cleanup commands with the same safety model.

Who it is for

Klene is for people who want a cleanup tool that explains itself, shows its work, and stays conservative by default. Arch remains the most complete platform, and the newer Linux and Windows providers stay deliberately careful.
